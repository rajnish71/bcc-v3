03

Save Behaviour & Field States

SAVE MECHANISM

Explicit save · whole-profile

A single Save Changes button saves all edited fields across all sections simultaneously. No per-section save. No auto-save (risks accidental partial publishes to downstream modules).

Unsaved state: When any field changes, the status text switches from "All changes saved" (grey italic) to "Unsaved changes" (amber, weight 500). The Save Changes button gains a subtle pulse on the first change only.

Sticky mini-bar: After 200px scroll, a sticky mini-bar appears at the top of the viewport — avatar thumbnail + member name + Save Changes button. Disappears when the user scrolls back to the profile header. Never two Save buttons in the same scroll viewport simultaneously.

Sticky bar occupies the top of the main content viewport. Save Changes button is the only interactive element in the bar. The bar disappears when the user scrolls back to the full profile header — the two Save targets are never simultaneously visible.
REQUIRED FIELDS
First Name · Surname · Email (system) · Membership Tier (system). All other fields optional. Save is never blocked by optional fields — even empty profiles save successfully.
VALIDATION TIMING
Validate on blur (field exit), not on keystroke. Error clears as soon as the value becomes valid. Never show multiple errors simultaneously — one per field. No success toast on save — the "All changes saved" status text is sufficient feedback.
DOWNSTREAM IMPACT
Save triggers a platform-wide profile cache invalidation. Downstream consumers (Public Profile, Showcase credits, Upload Module, Membership Card, Member Directory) receive updated data on next render.
