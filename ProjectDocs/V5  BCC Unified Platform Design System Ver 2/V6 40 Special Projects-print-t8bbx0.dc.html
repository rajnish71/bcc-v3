Rich Text Support

Photography Rule — Responsive

Natural aspect ratios are preserved at all breakpoints without exception. Justified rows collapse to column stacks on narrower viewports but photographs never crop. The only exception is the RelatedProjects card cover image — this uses object-fit: cover at the card thumbnail only, per editorial card convention.

Four data-empty conditions. Three result in section suppression — no DOM node, no heading, no placeholder. One requires an inline message. Empty states are always data conditions, never design conditions.

Data Schema — Required Fields

Design-level schema definition. All fields required unless marked optional. Implementation detail is defined separately in the engineering specification. Route is /api/v1/projects/{slug}.

Project Object
project {
  title         string   // H1, section headings
  slug          string   // route param, related exclude
  heroImage     url      // ImageKit CDN · natural ratio
  description   string   // hero overlay copy
  about         string?  // optional · rich text
  subjectLabel  string?  // stat 3 label (Species / etc)
  stats {
    photoCount        int
    contributorCount  int
    subjectCount      int?  // optional
    lastUpdated       string
  }
}
Contributors + Gallery
contributor {
  id            int
  displayName   string
  username      string   // → /photographers/[username]
  avatar        url?     // optional · initials fallback
  shortBio      string?  // optional · card bio
  contributionCount int
}

photo {
  id            int      // → /photos/[id]
  imageUrl      url      // ImageKit CDN
  aspectRatio   float    // from EXIF · natural ratio
  title         string?
  photographer  { username, displayName }
}
Related Projects
relatedProject {
  title         string
  slug          string   // → /projects/[slug]
  heroImage     url      // card cover · object-fit cover
  photoCount    int
}
