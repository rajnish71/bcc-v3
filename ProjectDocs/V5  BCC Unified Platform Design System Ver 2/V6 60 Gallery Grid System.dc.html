Canonical Architecture

Four-layer separation: Engine → Layout → Card → Theme. Each layer has one responsibility. Layers do not reach past their immediate neighbour. Public and admin contexts share the engine and layout layers; they differ at the Card layer only.

GalleryGridSystem · Canonical Architecture · V6 60
Layer 1
GalleryEngine
Shared core. Context-agnostic. Owns data flow, state, and pagination only.
photos[ ] layout context filters pagination loading onPhotoClick

LayoutEngine
Routes photos[ ] to the correct layout algorithm. Computes row geometry for Justified. Provides slot positions for Masonry, Modular, Editorial, Metro.

Layer 3
PhotoCard
The only layer that differs by context. Same interface, different visual treatment. Public: photography-first. Admin: productivity-first.

Deprecation Map

One layout deprecated. No breaking changes. Existing consumers using Magazine layout receive Editorial automatically. Migration is silent — no code change required at the call site.

Deprecated Magazine layout → auto-migrates to Editorial
layout="magazine"
Magazine Layout
deprecated · do not use
→
layout="editorial"
Editorial Layout
production · use this
LayoutEngine.resolve(layout):
if layout === "magazine":
console.warn(DEPRECATION_MSG)
return "editorial"
return layout

No call site change needed.
Deprecation warning in dev only.
Why Deprecated
Magazine was an early layout concept that described a large lead photo with text beside it. Editorial captures this intent more precisely and extends it with a secondary row. Magazine never reached production deployment.
Migration Steps
1. No code change at call site
2. LayoutEngine silently remaps
3. Dev console warning logged
4. Replace layout="magazine"
with layout="editorial"
in source when convenient
5. Warning disappears
Removal Timeline
Magazine alias removed at V7 engine upgrade only. Not before. V6 carries the alias for the full V6 lifecycle. No forced migration sprint required. Teams migrate at their own pace.
