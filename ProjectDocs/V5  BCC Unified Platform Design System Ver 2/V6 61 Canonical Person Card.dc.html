Purpose & Scope

The Person Card is the atomic unit of member identity presentation. It is the single contract between a person's profile data and the user interface — in every context, for every role, across the entire platform.

Governing Principle

One card. Everywhere.

There is no "Directory card", "Hub card", or "Admin card". There is ONE Person Card with four variants. Variant selection is driven by context and layout density — never by page, feature, or role. Any divergence requires design authority review.

Person-first. Data-aware.

The card always leads with the person — name and avatar — never with membership class or status. Constitutional data (membership class, recognition) is secondary, displayed below identity. This is not optional. MEM-006 prohibits class promotion on person presentation surfaces.

MEM-006 Single Active Rule
A member may hold at most ONE active recognition at any time. The Person Card never stacks two recognition badges. When a Governance Recognition is active, Automatic Recognition is deactivated. The card shows only recognition.active_title — never historical recognition.

Avatar — Initials Colour Assignment

When no avatar photo is set, the system renders initials on a deterministic background colour drawn from a fixed palette. Colour is derived from user_id modulo 7. Never from membership class — colour is identity-only, not status.
