Props API

Both components accept structured props. Designed for future database integration via Admin and Member Hub. All string fields will map to CMS-managed content. No page-specific logic is baked in.

DB Schema notes
Hero image is stored in the CMS pages table with fields: hero_image_url, hero_image_width, hero_image_height, hero_image_alt. Aspect ratio is derived from width/height — never hardcoded. The 16:9 CMS filter rejects uploads that do not meet ratio tolerance ±0.05.
Admin integration notes
No Admin UI is defined here. Administrative editing will use minimal functional forms, specified separately. Section Header variant ("standard" | "centered") will be set per-page in CMS page config. All text fields are editable via Admin. Hero image is administrator-curated — members cannot set page hero images.
