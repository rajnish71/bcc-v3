Rail Rule

04

Permission Behaviour

Navigation is role aware on two independent axes. Design authority only — no resolution logic is specified here; that belongs to the engineering RBAC layer.

Axis 1 — Phase Activation
Photography School and Contests are visible to every role at reduced emphasis with a SOON marker until their module ships. This is a platform-phase state, not a role state — it never hides, only mutes. Consistent with V6 00 §07's public-nav "Coming Soon" rule.
Axis 2 — Role Elevation
A fifth rail group appears only once a member holds an elevated RBAC role. It is additive — the base STUDIO / GROW / CLUB / ACCOUNT set is identical for every authenticated member regardless of role.

05

Responsive Behaviour

Three fixed breakpoints, matching V6 09's Hub shell. The Hub never stacks navigation above content on mobile — horizontal bars are preferred over a collapsing drawer.

≤ 640px HubMobileBar Fixed bottom bar, 5 slots, raised gold Upload centre action. "Club" and "More" open bottom sheets.

08

Annotation Index

① Logo lockup — 26px square mark + "Member Hub" wordmark, top of rail. Not the public SiteHeader logo; signals the Hub context distinctly.
② Group label — 11px / 0.18em / uppercase / ink-400. Never clickable. Fixed order across every role: STUDIO, GROW, CLUB, ACCOUNT.
③ Active indicator — 6px gold dot, one per rail maximum. Never a filled background or left border.
④ Coming Soon marker — mono "SOON" chip, ink-400 on surface-2. Applies to Photography School and Contests regardless of role.
⑤ Elevated group — fifth group, appended below ACCOUNT, present only for Editor / Moderator / Admin sessions per §04.
⑥ Sign Out footer — pinned to rail bottom via flex margin-top:auto, always visible without scrolling the rail itself.
09

Component Tree

HubNavigation is a single shared authority mounted once per Hub layout, rendering one of three surfaces by breakpoint. Every Hub page (Home, Portfolio, Upload Studio, Membership & Billing…) imports HubLayout — never HubNavigation directly.

Data contract
HubNavigation reads {member.role, member.entitlements, platform.phaseFlags} and a single static hubNav[] tree (groups → items). No page passes its own nav data.
Single source rule
hubNav[] is defined once. Rail, Tabs, and MobileBar all read the same structure — they differ only in presentation, never in content or order.
Auth gate
HubLayout — and therefore HubNavigation — requires role ≥ "member". Unauthenticated or under-authorised access never reaches this component; it redirects to /auth/signin per V6 00 §08.
