BEHAVIOURAL RULES
Independent saves: Each of the 3 sections saves independently. No global save. No auto-save.

Success feedback: Inline text only — "Name updated." / "Password updated." No toast, no redirect.

Validation: On blur (field exit), not on keystroke. Error clears when value becomes valid.

Email change: Two-step. Send → verify. Current email stays active until link is clicked. Resend timer-gated at 60 s.

Gold CTA rule: Send Verification Link is NOT gold. Gold = save action only. (Section 2 has no gold CTA by design.)
AUTH GATE
Route /hub/account-settings requires authenticated session with role ≥ USER. Unauthenticated redirect → /auth/signin with return-url preserved.
USERNAME LOCK
Username is permanent — MEM-007 identity lock. It is displayed as a read-only reference below the Display Name Preview. It is NEVER an editable field on any page of the platform.
DOWNSTREAM IMPACT
Name changes propagate to: Public Profile, Membership Card, Member Directory, Showcase credits, Upload Module, Consent Form. Changes are reflected on next render of downstream consumers.
