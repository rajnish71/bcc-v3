§
Governing Rule — Every Legacy Field Must Be Preserved

Every profile field that exists in the legacy system must be preserved in V3. This rule applies regardless of whether the field is currently populated, currently empty, displayed publicly, displayed in the hub, currently unused, or planned for a future module. Presentation does not determine preservation. The V3 database is the complete system of record. The UI is only one consumer of that data.

Legacy Profile
Reference material only. Tells us what information historically existed. Not a design authority.
Member Hub Profile Editor
Editing interface. Allows members to maintain their complete profile. Contains many fields that never appear publicly — address, personal data, internal membership info, privacy preferences, admin metadata.
Public Photographer Profile
Presentation layer only. Consumes profile information from the hub. Intentionally exposes only selected public fields. Must never define the underlying schema.
